    What I like to do is use the stylistic class in addition to a unique ID. I prefer an ID because it allows us a unique way to call each button in JS, but also stick to a consistent naming scheme: button_buttonTYPE.

    Now let's apply those variables instead of the colors that are defined in our CSS. I'm taking this time to do even more cleanup: organizing the ORDER of the CSS lines so that they're the same across all the buttons, fixing the Call to Action button so it has the same border properties.     Button Mixin

    
@mixin button($text-color, $bg-color, $if-disabled) {
    @extend %button;
    color: $text-color;
    background-color: $bg-color;
    @if $if-disabled == false {
        box-shadow: 0px 8px 0px 0px transparentize($bg-color,.75);
        border-right: 1px solid darken($bg-color,10%);
        border-bottom: 1px solid darken($bg-color,10%);
        text-shadow: 0px 1px 1px darken($bg-color,30%);
    }
    @else if $if-disabled == true {
        // including these because button_disabled is _added_ to classes, instead of toggled, so the primary/secondary/etc styles still apply, so we have to remove them
        box-shadow: none;
        border: 0;
        text-shadow: none;
    }
    &:active, &:hover {
        background-color: darken($bg-color,10%);
    }
}

// To Include
.button_primary {
    @include button($button_primary_color,$button_primary_background,false); // grab all the above styles
}

    The box-shadow, text-shadow, border, and hover background color states are actually all redundant code as well. If we wanted, we could pull these out into a mixin.

    This mixin looks for three variables: background color, text color, and a true/false value for this $if-disabled state. Then it extends the button placeholder, applies the text and background color variables that we pass in, and then IF the if-disabled check is false, it applies the border and shadows. If the check is true, which it will be for our .button_disabled, it reverts the border and shadow properties. Instead of just not applying them, it reverts them, because we are _adding_ the disabled class instead of toggling it.

    For each button class, then, we just say .button_primary, include mixin button, and pass in the relevant color variables, and unless it's the disabled button, false.

    This abstracts out the code even more, and makes applying a new button style, like say, .button_magic, really easy. We'd just add 2 variables: $button_magic_color, $button_magic_background, and then class .button_magic and include the mixin.

    
$button_magic_color: $white;
$button_magic_background: $green;

.button_magic {
    @include button($button_magic_color,$button_magic_background);
}
